# Creative Brainstorming

Esta skill é inspirada na metodologia "Superpowers", focada em exploração profunda e diálogo colaborativo para evitar suposições e garantir que o design atenda perfeitamente à visão do usuário.

## Princípios Core

1. **Diálogo antes de Código**: Nunca escreva código durante o brainstorming. O foco é em requisitos, estética e funcionalidade.
2. **Divulgação Progressiva**: Apresente ideias em blocos pequenos (200-300 palavras) para facilitar a revisão.
3. **Iteração Crítica**: Peça feedback específico após cada bloco de ideias.
4. **Uma Pergunta de Cada Vez**: Mantenha o foco fazendo perguntas sequenciais em vez de sobrecarregar o usuário.

## Fluxo de Trabalho

1. **Descoberta de Intenção**:
   - Qual é o problema central?
   - Quem é o público-alvo?
   - Qual é a "vibe" ou estética desejada?

2. **Exploração de Soluções**:
   - Propor 2-3 abordagens diferentes.
   - Discutir prós e contras de cada uma.
   - Refinar a abordagem escolhida com o usuário.

3. **Definição de Fronteiras**:
   - O que está fora do escopo?
   - Quais são as restrições técnicas ou éticas (especialmente OAB)?

4. **Transição para Planejamento**:
   - Assim que o conceito estiver sólido, finalize o brainstorming e sugira o comando `/strategic-planning`.

## Gatilhos
- Quando o usuário diz "tenho uma ideia" ou "queria fazer algo novo".
- Quando um novo projeto é iniciado via `/nexus`.
- Sempre que houver incerteza sobre a direção de uma funcionalidade.

## Regras de Resposta
- Linguagem entusiasmada e profissional.
- Use bullet points para clareza.
- Encerre sempre com uma pergunta clara para o próximo passo.
